# 🌐 Day 1: AWS 글로벌 인프라 기초 및 격리된 VPC 네트워크 아키텍처 설계

1일차에는 AWS의 물리적/논리적 인프라 단위를 이해하고, 백엔드 시스템의 안전한 기반이 되는 독립된 가상 네트워크 환경인 **Amazon VPC**를 직접 설계했습니다. 특히, 서비스의 안전성을 높이기 위해 **보안 그룹(Security Group)**과 **네트워크 ACL(nACL)**을 활용한 **다중 방어 계층(Defense in Depth)**을 구축하고 검증하는 데 집중했습니다.

---

## 📂 디렉토리 구조 및 실습 내용

### 1. [01_AWS_GlobalInfrastructure_VPC](./01_AWS_GlobalInfrastructure_VPC/)
- **핵심 개념:** 리전(Region) 선택 기준, 가용 영역(AZ), CIDR 블록 표기법 및 IP 할당 방식
- **실습 내용:**
  * **VPC 및 서브넷 설계:** 가상 네트워크망(`VPC`)을 생성하고, CIDR 범위를 쪼개어 외부 통신이 가능한 **Public Subnet**과 내부망으로 격리된 **Private Subnet** 환경 구성
  * **공인 IP 할당 방식 분석:** EC2 인스턴스에 퍼블릭 IP를 매핑하는 동적 방식과 고정형 자원인 탄력적 IP(EIP)의 차이점 학습
  * **NAT 게이트웨이 실습:** Private Subnet 내부에 배치된 가상 머신(`vm01`)이 외부 인터넷과 안전하게 아웃바운드 통신을 주고받을 수 있도록 라우팅 테이블 및 NAT 게이트웨이 연동

### 2. [02_VPC_Security_nACL_SG](./02_VPC_Security_nACL_SG/)
- **핵심 개념:** 인바운드(Inbound)/아웃바운드(Outbound) 트래픽 흐름 제어, 다중 방어 계층(Multi-layered Security)
- **실습 내용:**
  * **보안 그룹(Security Group) 제어:** 인스턴스 레벨의 방화벽인 `my-private-sg`를 생성하고, Stateful 특성을 이해하며 허용(Allow) 규칙 설정 규칙 마스터
  * **네트워크 ACL(nACL) 적용:** 서브넷 레벨의 Stateless 방화벽을 적용하여, 특정 IP 대역이나 포트를 명시적으로 차단(Deny)하는 제어 실습

## 🔍 핵심 이론 및 실습 비교 분석

### 🛡️ 보안 그룹(SG) vs 네트워크 ACL(nACL) 한눈에 보기

| 비교 항목 | 보안 그룹 (Security Group) | 네트워크 ACL (Network ACL) |
| :--- | :--- | :--- |
| **적용 범위** | **인스턴스 레벨** (EC2 각각에 설정) | **서브넷 레벨** (서브넷 전체에 설정) |
| **동작 방식** | **Stateful** (들어온 신호는 나갈 때 무조건 허용) | **Stateless** (들어올 때, 나갈 때 규칙을 각각 검사) |
| **규칙 특징** | **허용(Allow) 규칙**만 정의 가능 | **허용(Allow) 및 차단(Deny) 규칙** 모두 가능 |
| **우선순위** | 모든 규칙을 종합하여 평가 | 규칙 번호 순서대로 순차적 평가 (우선순위 존재) |

## 💡 학습 회고
- **네트워크 격리의 필요성 체득:** 모든 인프라 자원을 인터넷에 노출하는 것이 아니라, VPC 내에서 Public과 Private 영역을 명확히 분리하고 라우팅 테이블을 직접 제어해 보며 백엔드 보안의 출발점을 배울 수 있었습니다.
- **다중 방어 아키텍처 이해:** 웹 서버나 DB로 향하는 트래픽을 서브넷 관문(nACL)에서 1차로 필터링하고, 인스턴스 바로 앞(보안 그룹)에서 2차로 촘촘하게 한 번 더 방어하는 구조를 직접 실습하며 클라우드 보안 아키텍처의 중요성을 체감했습니다.
